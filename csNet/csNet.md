

# csNet 开发文档

`csNet` 是一个专为局域网双端节点设计的极简、高可用 P2P 通信底层库。它将复杂的网络重连、心跳保活、连接冲突仲裁等细节彻底封装，对外只暴露纯粹的“发送”与“接收”，让你可以用写单机代码的心态写局域网联动程序。

## ✨ 核心特性

- **绝对非阻塞与自动丢弃策略**：`Send()` 方法永远不会阻塞，哪怕断网数年、每秒狂发一万次也瞬间返回。内部采用环形缓冲区，当积压消息超过 100 条时，**自动丢弃最旧的消息，保留最新的 100 条**，完美适配高频数据流（如鼠标坐标同步、游戏状态同步）。
- **丝滑重连与假死检测**：内置严格写超时的应用层心跳，拔网线、杀进程、重启电脑等引发的“半打开”假连接会在数秒内被底层主动斩断并自动重连，应用层无感。
- **防错位仲裁算法**：当两端同时重启并同时尝试连接对方时，底层通过比对 IP 自动裁决，瞬间掐断错误的交叉连接，保证两端物理上使用同一条管道，彻底杜绝“发了消息对面收不到”的 bug。
- **断线最后一条补发**：在连接断开的瞬间你发出的最后一条消息，会被特殊暂存，在重连成功后的第一毫秒自动补发。
- **纯异步 Channel 驱动**：接收数据直接通过 Go Channel 遍历，极其适合接入 GUI 框架、Web 接口或游戏主循环。

## 🚀 快速开始

### 1. 引入库

将 `csNet` 文件夹放入你的项目目录，假设你的 `go.mod` 叫 `myapp`：

```go
import "myapp/csNet"
```

### 2. 最小化示例

只需 4 步，即可在后台建立起坚如磐石的通信通道：

```go
package main
import (
    "fmt"
    "myapp/csNet"
)
func main() {
    // 1. 初始化平台底层（处理Windows乱码、Linux崩溃信号等，建议调用一次）
    csNet.Init()
    // 2. 配置参数
    cfg := csNet.Config{
        PeerIP: "192.168.1.100", // 对方IP
        Port:   "9527",          // 通信端口（两端必须一致）
    }
    // 3. 创建实例（非阻塞，后台自动开始拨号/监听/心跳）
    peer := csNet.New(cfg)
    defer peer.Close() // 程序退出时清理资源
    // 4. 阻塞等待首次连接成功
    <-peer.Connected
    fmt.Println("连接已建立！")
    // 5. 收发数据
    go func() {
        for msg := range peer.Recv {
            fmt.Printf("收到: %s\n", msg)
        }
    }()
    // 即使现在拔掉网线，这行代码也绝不会卡死
    peer.Send("Hello World!")

    // 阻塞主线程
    select {}
}
```

## 📖 API 参考

### `csNet.Init()`

初始化运行环境。在 Windows 上会自动设置控制台为 UTF-8 编码，在 Linux 上会屏蔽 `SIGPIPE` 导致的程序崩溃。
**建议**：在 `main` 函数入口处调用一次即可。

### `csNet.Config` 结构体

创建连接时需要传入的配置项，时间参数均可选（不填则使用合理的默认值）：
| 字段 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| `PeerIP` | `string` | **必填**，目标电脑的局域网 IP | 无 |
| `Port` | `string` | **必填**，通信使用的端口（两端必须一致） | 无 |
| `DialTimeout` | `time.Duration` | 主动拨号单次超时时间 | `1s` |
| `AcceptTimeout`| `time.Duration` | 被动监听单次超时时间 | `1s` |
| `HeartbeatInterval`| `time.Duration`| 心跳包发送间隔 | `3s` |
| `HeartbeatTimeout` | `time.Duration`| 心跳包写入超时（超时即判定断线）| `3s` |
| `ReconnectDelay` | `time.Duration`| 断线后重新发起连接的间隔 | `1s` |

### `csNet.New(cfg Config) *Peer`

创建并启动一个 P2P 网络实例。此函数是非阻塞的，调用后会在后台自动开启连接竞速。

### `*Peer.Send(data string)`

向对方发送数据。
**核心机制**：本方法是**绝对非阻塞**的，内部维护了一个容量为 100 的环形缓冲区。如果当前网络畅通，数据会立即发送；如果断网导致积压超过 100 条，**最老的数据会被自动丢弃，以腾出空间给最新来的数据**。本方法永远不会返回 error。

### `*Peer.Recv <-chan string`

只读 Channel。用于接收对方发来的业务数据。
**生命周期**：当连接断开正在重连时，此 Channel 会被挂起（**不会关闭**），直到重连成功后继续流入新数据。只有在调用 `Peer.Close()` 彻底销毁实例时，此 Channel 才会关闭。

### `*Peer.Connected <-chan struct{}`

信号 Channel。当底层**首次**成功建立连接时，此 Channel 会被 `close()`。适用于在程序启动时阻塞等待网络就绪。注意：后续的断线重连不会再次触发此 Channel。

### `*Peer.Close()`

彻底关闭底层网络，释放所有协程和端口占用。

### `csNet.SetupFirewall(port string)` / `csNet.CleanupFirewall(port string)`

**仅 Windows 有效**。尝试通过 `netsh` 添加/删除防火墙入站规则。如果当前终端没有管理员权限，函数会静默跳过，不会报错。

## ⚠️ 重要设计哲学与注意点

阅读以下内容可以避免 90% 的误用：

### 1. 丢弃策略的真实意图

为什么积压超过 100 条要丢掉旧数据，而不是无限缓存？
在局域网联动（如同步鼠标位置、游戏帧数据）场景中，如果断网了 10 秒，积压了 1000 条坐标数据，重连后把这 1000 条发给对方是没有意义的（对方只需要最新的坐标）。`csNet` 的策略是：**尽最大努力保留最近的状态，绝不拖慢调用方的节奏**。

### 2. 断线重连时的“抢救”机制

如果在连接断开的瞬间，`Send()` 刚好把一条数据放入了队列，这条数据会成为 `Pending` 状态。当底层重连成功后，**只会补发这最后一条 Pending 消息，并清空队列里的其他历史积压**。这也是为了防止重连后出现“消息轰炸”。

### 3. 保留字冲突

底层使用 `_SC_NET_PING_` 作为心跳探测包（底层自动过滤，不会出现在你的 `Recv` 通道里）。**请勿将这个字符串作为业务消息发送**，否则会被底层静默吞掉。

### 4. 不要手动操作 Channel

`peer.Recv` 是由库内部严格管理的，千万不要在业务代码中手动 `close(peer.Recv)`，否则会引发 panic。
